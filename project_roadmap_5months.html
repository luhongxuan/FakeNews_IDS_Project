5 個月專題開發路線圖

假訊息查核 × 擴散追蹤 × 干預模擬平台 ｜ 3 人團隊
第 1 個月 研究規劃 & 資料探索 週 1－4 ▶

把所有理想用「能不能真的做出來」來驗證，確認技術可行性，選定資料集，寫好文件。

核心任務
下載並探索 PHEME、FakeNewsNet 資料集，了解欄位結構與限制
試打 Google Fact Check Tools API，確認查詢效果與回傳格式
試用 GDELT API 做新聞事件查詢，評估資料量與品質
定義系統架構：前後端分工、API 設計、資料庫結構
確定三人分工：後端 / 圖分析 / 前端視覺化
使用工具
Python (pandas) Jupyter Notebook Google Fact Check API GDELT API Figma（系統設計稿）
建議分工
A — 資料集探索 + API 評估 B — 系統架構設計 C — UI wireframe
🎯里程碑：能用 Fact Check API 查到一個 claim 的完整回傳結果
第 2 個月 後端核心 MVP 週 5－8 ▶

做出可以接受使用者輸入、完成查核搜尋、抓取新聞資料的後端 pipeline。

核心任務
實作 claim 輸入 → Fact Check API 查詢 → 結構化回傳
實作 GDELT 新聞抓取 → 去重 → 時間排序 pipeline
設計並建立資料庫（PostgreSQL 或 SQLite）儲存查核結果快取
用 sentence-transformers 做基礎 Claim Clustering（相似 claim 合併）
建立 FastAPI 後端，定義 /search、/timeline、/graph 三個端點
使用工具
FastAPI PostgreSQL / SQLite sentence-transformers requests / httpx Pydantic
建議分工
A — Fact Check + GDELT pipeline B — FastAPI 架設 + DB schema C — Claim Clustering
🎯里程碑：輸入「COVID vaccine DNA」，後端回傳查核結果 + 5 篇相關新聞 + 時間排序
第 3 個月 圖分析 & 擴散模型 週 9－12 ▶

把新聞與來源關係轉成圖結構，分析關鍵節點，跑擴散模擬與干預策略比較。

核心任務
從 PHEME 資料集建構傳播圖（節點 = 帳號/媒體，邊 = 轉貼/引用）
用 NetworkX 計算 degree / betweenness / PageRank，找出關鍵放大節點
用 NDlib 跑 SIR 模型模擬假訊息擴散過程
實作 4 種干預策略：random / degree / betweenness / greedy，比較效果
設計「干預前後」的比較輸出格式，串接 API
使用工具
NetworkX NDlib NumPy / SciPy matplotlib（初步驗證圖）
建議分工
A — 圖建構 + centrality 分析 B — NDlib 擴散模擬 C — 干預策略實作 + 比較
🎯里程碑：能對一個 PHEME 事件跑出 4 種策略的干預效果比較圖
第 4 個月 前端視覺化 & 整合 週 13－17 ▶

把後端所有結果呈現成可操作的前端介面，完成三層完整 demo 流程。

核心任務
實作查核入口介面：輸入框 → 查核結果卡 → 可信度量尺
實作新聞時間線：橫軸時間、縱軸平台、事件點可點擊展開
用 Cytoscape.js 或 D3.js 實作互動式擴散圖（點節點看詳細資訊）
實作干預模擬面板：選策略 → 選 k 值 → 顯示干預前後擴散量比較
串接所有 FastAPI 端點，完成 end-to-end 流程
加入 PDF 報告匯出功能
使用工具
React Cytoscape.js / D3.js Recharts（時間線 / 比較圖） TailwindCSS jsPDF（報告匯出）
建議分工
A — 查核入口 + 時間線 B — 擴散圖視覺化 C — 干預模擬面板 + 報告匯出
🎯里程碑：完整 demo 跑通一個案例（從輸入 → 查核 → 擴散圖 → 干預比較）
第 5 個月 優化、文件與發表準備 週 18－20 ▶

把系統從「能跑」變成「能說服人」，準備好 demo 腳本、論文、簡報。

核心任務
選定 3 個 case study（一個已知假訊息事件 replay）做 demo 腳本
UI 細節優化：載入速度、空狀態提示、錯誤處理
寫專題報告：動機 → 架構 → 實驗設計 → 結果 → 未來展望
製作發表簡報：突出三層架構亮點，準備 live demo
部署到雲端（Render / Vercel / Railway），讓老師可以線上試用
使用工具
Render / Railway（後端部署） Vercel（前端部署） Docker（環境打包） Notion / Overleaf（報告）
建議分工
A — 部署 + case study B — 報告撰寫 C — 簡報製作
🎯里程碑：系統上線、demo 腳本跑順、報告初稿完成
